# サプライチェーン方針

## 基本方針

- `Cargo.lock`、`fuzz/Cargo.lock`、`flake.lock` をコミット対象にします。
- Rust依存は `cargo-deny` で既知問題、ライセンス、取得元、重複を検査します。
- CVEの一時例外は `scripts/cve-audit-baseline.json` に期限と理由を付けます。
- Nix配布物を `nix flake check` と `nix build .#default` で検証します。
- 新規依存には利用箇所、代替案、ライセンス、保守状況を変更説明へ記載します。

理由: VPNクライアントはネットワーク境界と認証情報を扱うため、依存の由来と既知脆弱性を再現可能に監査する必要があります。

不採用理由: 版の範囲だけを宣言して依存版固定ファイルを持たないと、同じコミットでも検証対象が変化します。

## ライセンス

本プロジェクト自身のライセンスはApache-2.0であり、リポジトリ直下の`LICENSE`を正とします。依存ライセンスの許可リストは `deny.toml` を正とします。例外を追加する場合は、配布形態との互換性を確認して理由を同じ変更に記録します。

理由: 将来の携帯端末向け配布と外部関数境界の成果物を含めて、再配布条件を追跡するためです。

不採用理由: ビルドに成功することはライセンス互換性を保証しません。
