# スコープ

## 目的

SSTPのプロトコル処理を、オペレーティングシステム固有のVPN接続仕様、利用者画面、プロセス制御から分離して実装します。純粋な状態機械を中心に置き、各実行環境では必要な入出力だけを接続部としてつなぎます。

理由: 性能劣化の所在を計測可能にし、オペレーティングシステム差分をプロトコル実装へ漏らさず、同じ試験値を全実行環境で再利用するためです。

不採用理由: オペレーティングシステムごとに独立したSSTP実装を持つと、不具合修正と仕様追従が分岐し、性能差の原因も比較できません。

## 対象

- SSTP 1.0クライアントの通信形式の符号化・復号と制御メッセージ
- HTTP/TLS上のSSTPトランスポート
- PPPフレームの受け渡しとネゴシエーション境界
- Linux TUN、Android `VpnService`、Apple Network Extensionとの接続
- Flutterから利用するための狭い外部関数境界と接続仕様
- 相互運用、異常系、性能、長時間接続の試験用実行基盤

理由: これらが利用可能なクライアントを形成する最小の責務集合だからです。

不採用理由: 利用者画面を先に含めると、通信中核の正しさと性能を利用者画面のライフサイクルから独立して検証できません。
