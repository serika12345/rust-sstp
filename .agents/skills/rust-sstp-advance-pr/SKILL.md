# PRを進める

1. 現在のブランチとPRを確認する。
2. 編集前に完了条件、含む内容、含まない内容、検証、停止条件を記す。
3. 一貫した一つの縦切りを、試験から先に実装する。
4. 無関係な整形、改名、依存更新、推測に基づく抽象化を避ける。
5. `nix develop -c ./scripts/verify` を実行し、正確な結果を要約する。
6. プロジェクト記録を更新する。ただし明示的な許可なしにPRの作成、送信、統合を行わない。
