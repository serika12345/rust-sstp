# 既存ブランチの作業を再開する

1. ブランチ、状態、差分、最近の履歴、`TODO.md`、`docs/design-todo.md`、`docs/progress.md`を確認する。
2. 既存変更は利用者の所有物として扱い、破棄、初期化、上書きをしない。
3. 証跡から意図した作業区切りを推定し、不確実性を明示する。
4. 編集前に有用な最小範囲の試験を行う。
5. テスト駆動開発による一貫した最小増分を進めて検証する。
6. 競合する変更から異なる完成像が考えられる場合は停止し、利用者の指示を求める。
