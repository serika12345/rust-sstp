# 確認済み作業を統合可能な状態にする

1. コミット、送信、変更提案、統合について利用者の明示的な許可を確認する。
2. 現在の作業区切りとすべての確認指摘を読み直す。
3. 指摘が修正済みか、利用者に明示的に受け入れられたことを確認する。
4. 状態を把握した作業ツリーで `nix develop -c ./scripts/verify` を実行する。
5. 検証済みの結果だけを `TODO.md` と `docs/progress.md` へ反映する。
6. ブランチ保護の回避、強制送信、失敗または不明な検査結果での統合を行わない。
