# 現在の作業区切りを確認する

1. 作業区切りを特定し、最終ファイルだけでなく差分を確認する。
2. プロトコルの正しさ、不正状態の構築、バッファー上限、秘密情報の露出、取り消しと流量制御、オペレーティングシステム境界からの漏出を優先する。
3. 正常系の網羅だけでなく、失敗形態を試験できているか確認する。
4. 可能なら対象を絞った検査と `nix develop -c ./scripts/verify-quick` を実行する。
5. 指摘を重大度順に、ファイルと行の参照付きで先に報告する。
6. 後の作業区切りで対処できるという理由だけで、指摘の編集、コミット、統合、無視を行わない。
