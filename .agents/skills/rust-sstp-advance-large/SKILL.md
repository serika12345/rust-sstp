# 現在の作業区切りを完了まで進める

1. 作業区切りに関係する入力をすべて読み、完了確認一覧を定める。
2. テスト駆動開発による小さな単位で作業し、進行中の単位を一つだけに保つ。
3. 各単位の後に対象を絞った検証を行い、完了を宣言する前に `nix develop -c ./scripts/verify` を実行する。
4. 作業区切り内の問題だけを解決し、隣接作業は範囲を広げず記録する。
5. `TODO.md`、`docs/design-todo.md`、`docs/progress.md`を検証済みの実態と一致させる。
6. 利用者が明示的に依頼しない限り、コミット、送信、統合、外部サービス呼び出しを行わない。
