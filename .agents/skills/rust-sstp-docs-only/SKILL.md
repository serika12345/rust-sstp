# 文書だけを整合させる

1. コードと試験を証跡としてのみ読む。
2. `README.md`、`DESIGN_PLAN.md`、`TODO.md`、`docs/design-todo.md`、`docs/progress.md`を整合させる。
3. 計画済み、未解決、実装済み、検証済みの区別を保つ。
4. 重要な設計判断には理由と不採用理由を必ず含める。
5. Rustソースコード、依存、依存版固定ファイル、処理手順、生成物を変更しない。
6. 作業空間を変更しない文章・構造検査を実行する。
