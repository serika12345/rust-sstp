# 作業一覧

実装順序の唯一の正です。一度に進行中にする変更単位の完了条件は一つだけとし、未確定判断は `docs/design-todo.md`、完了証跡は `docs/progress.md` に置きます。

## H0 初期試験基盤

- [x] Nix Flakeとdirenvで正規開発環境を定義する
- [x] Rust作業空間と最小クレート境界を作る
- [x] プロトコル層と接続処理層を入出力なしの純粋層として機械検査する
- [x] 整形、静的検査、試験、供給網、Nix配布物の検証を単一入口へ統合する
- [x] スコープ、設計、規約、依存方針、進捗を文書化する

### 変更単位の完了条件 H0

- 完了条件: `./scripts/verify` がNix環境で成功する
- 含む: 開発用試験基盤、品質確認、最初の純粋な状態遷移
- 含まない: 実ネットワーク接続、TLS、HTTP、PPP、オペレーティングシステムのVPN接続仕様、外部関数境界、利用者画面
- 検証: 整形、構成、領域型、`cargo-deny`、CVE、Clippy、試験、Nix構築
- 停止条件: Nixで同じ結果を再現できない、またはクレート依存方向を破る必要が生じる
