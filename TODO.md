# 作業一覧

実装順序の唯一の正です。上から順に最初の未完了項目だけを進行中のPRとします。未確定判断は `docs/design-todo.md`、完了証跡は `docs/progress.md` に置きます。

## PR単位の共通完了条件

各チェック項目は一つのPRです。個別記載に加えて、次をすべて満たした場合だけ完了とします。

- 新しい振る舞いは、先に追加した失敗試験によって確認されている。
- 正常系、境界値、分類された失敗を、PRに最も近い層で試験している。
- 公開APIはドメイン型を使い、`sstp-protocol` と `sstp-session` にI/O依存または `unsafe` を持ち込んでいない。
- 仕様根拠、重要な判断の理由と不採用理由、残した課題が対応する文書に記録されている。
- `./scripts/verify` がNix環境で成功し、結果が `docs/progress.md` に記録されている。
- 依存版固定ファイルを変更した場合は `./scripts/verify-supply-chain` も成功している。

理由: 一つのPRで確認する振る舞いと構造上の影響を限定し、単独で統合・差し戻しできるようにするためです。

不採用理由: 大見出し全体を一つのPRにすると、仕様判断、正常系、異常系、I/O接続が混在し、失敗原因と完了判定が不明確になるためです。

## H1 SSTP通信ヘッダーとデータパケット

- [x] H1-1 MS-SSTP 2.2.1の通信ヘッダーをドメイン型として符号化・復号する。
  - 完了条件: 版、制御・データ種別、12ビット長を持つ4バイトヘッダーが公式試験値と往復一致し、短すぎる入力、不支持版、4バイト未満の宣言長を分類して拒否する。
  - 含む: 仕様節と固定試験値の記録、送信時予約ビットのゼロ化、受信時予約ビットを無視する仕様の試験。
  - 含まない: ペイロード、制御メッセージ、TCPストリームの断片化処理。
  - 検証: `sstp-protocol` の単体試験と固定試験値。
  - 停止条件: MS-SSTPの規範記述と試験期待値が一致しない。

- [x] H1-2 SSTPデータパケットを一個の完全な入力から符号化・復号する。
  - 完了条件: データペイロードと宣言長の一致を検証し、最大4095バイトまで往復一致し、切り詰めと余剰を分類できる。
  - 含む: データパケットの所有権を表すドメイン型、最小長と最大長の境界試験。
  - 含まない: PPP内容の解釈、複数パケットの連続入力、メモリ再利用最適化。
  - 検証: MS-SSTP 2.2.3に基づく固定試験値と境界値試験。
  - 停止条件: 公開接続仕様でペイロードの意味を生バイト列のまま呼び出し側へ委ねる必要が生じる。

- [x] H1-3 断片化・結合された入力からSSTPパケットを逐次復号する。
  - 完了条件: 一個未満、一個、複数個の任意分割入力から同じパケット列を得て、未完了入力と不正入力を区別できる。
  - 含む: 上限付き蓄積、消費済み長の明示、分割位置の網羅試験。
  - 含まない: ソケット、TLS、非同期実行基盤。
  - 検証: すべての分割位置を生成する決定論的試験。
  - 停止条件: 復号器がI/Oまたは実行基盤を所有する必要が生じる。

- [ ] H1-4 通信ヘッダーとデータパケット復号器をファズ試験対象にする。
  - 完了条件: 任意入力で強制終了、範囲外参照、無制限確保を起こさず、成功値を再符号化すると同じ意味の値になる性質を検査できる。
  - 含む: 最小コーパス、実行時間を限定した通常検証と長時間検証の分離。
  - 含まない: 制御メッセージ、外部サーバー試験。
  - 検証: 通常検証の短時間走行と `./scripts/verify-nightly` の長時間走行。
  - 停止条件: 正規Nix環境でファズ試験を再現できない。

### H1完了条件

H1-1からH1-4が完了し、SSTPデータパケットの完全入力と連続入力を、I/Oなしで安全に符号化・復号できること。

理由: 制御メッセージより先に共通ヘッダー、長さ上限、逐次入力境界を固定すると、後続の解析器が同じ安全条件を再利用できるためです。

不採用理由: 最初から制御メッセージと状態機械まで一括実装すると、通信形式の失敗と状態判断の失敗を分離できないためです。

## H2 SSTP制御メッセージの符号化・復号

- [ ] H2-1 制御パケット共通部、接続要求、カプセル化プロトコル識別属性を符号化・復号する。
  - 完了条件: PPPを指定する接続要求が公式形式と往復一致し、メッセージ長、属性数、属性長の不一致を分類して拒否する。
  - 含む: MS-SSTPの該当節と固定試験値、接続要求に許される属性規則。
  - 含まない: 接続応答、状態遷移、PPP交渉。
  - 検証: 正常値、重複属性、切り詰め、過大宣言長の単体試験。
  - 停止条件: 属性規則の仕様根拠を確定できない。

- [ ] H2-2 接続肯定応答・否定応答と状態情報属性を符号化・復号する。
  - 完了条件: 応答種別と状態情報をドメイン型で表し、必須属性、重複、未知属性の受信方針が試験で固定される。
  - 含む: 未知属性の保持・無視・拒否の判断と、その理由および不採用理由。
  - 含まない: Crypto Binding計算、セッション遷移。
  - 検証: MS-SSTPの応答試験値と属性組合せ試験。
  - 停止条件: 未知属性の相互運用方針を仕様から決められない。

- [ ] H2-3 Crypto Binding要求・応答の通信形式を符号化・復号する。
  - 完了条件: nonce、ハッシュ方式、証明書ハッシュ、Compound MACを意味の異なるドメイン型で保持し、長さと方式の不整合を拒否する。
  - 含む: 接続完了メッセージと関連属性の通信形式。
  - 含まない: ハッシュ計算、鍵導出、秘密情報の生成。
  - 検証: SHA-256形式の固定試験値と異常長試験。
  - 停止条件: 通信形式と暗号計算の責務を分離できない。

- [ ] H2-4 中止、切断、切断確認、Echo要求・応答を符号化・復号する。
  - 完了条件: 各メッセージを列挙可能な型で区別し、属性禁止規則と長さを検証して往復一致する。
  - 含む: 全SSTP制御メッセージ種別の網羅性検査。
  - 含まない: どの状態で受理するかという判断、タイムアウト。
  - 検証: メッセージ種別ごとの固定試験値と未知種別試験。
  - 停止条件: 未実装メッセージを成功値として表現する必要が生じる。

- [ ] H2-5 全制御メッセージ復号器をファズ試験対象にする。
  - 完了条件: 任意入力と属性列で強制終了や無制限確保を起こさず、成功値の意味的な往復性を検査できる。
  - 含む: H1の逐次復号器との合成試験。
  - 含まない: セッション状態機械、実サーバー。
  - 検証: 短時間および夜間ファズ試験。
  - 停止条件: 解析器単体で入力上限を保証できない。

### H2完了条件

H2-1からH2-5が完了し、MS-SSTP 1.0の制御メッセージを状態判断なしで安全に符号化・復号できること。

理由: 通信形式と受理状態を分離し、同じ解析器を状態機械、試験用実行基盤、解析用途から利用できるようにするためです。

不採用理由: 解析中に状態遷移させると、途中失敗で状態だけが変化する危険があるためです。

## H3 SSTPセッション状態機械

- [ ] H3-1 外側通信路の確立から接続要求・応答までを型付き操作指示で表す。
  - 完了条件: 接続指示、通信路準備完了、接続肯定・否定応答の事象列から、送信・次状態・失敗理由が決定論的に得られる。
  - 含む: 状態ごとの受理可能事象と不正遷移の単体試験。
  - 含まない: 実TLS・HTTP、PPP、実時間。
  - 検証: 表形式の状態遷移試験。
  - 停止条件: 状態機械がソケットまたは非同期処理を所有する必要が生じる。

- [ ] H3-2 Echo、中止、正常切断、通信路切断の状態遷移を実装する。
  - 完了条件: 接続途中と接続後の各終了経路が、送信操作指示、最終状態、分類された終了理由を返す。
  - 含む: 相手起点と利用者起点の切断、重複終了事象の冪等性。
  - 含まない: 再接続方針、実ネットワーク。
  - 検証: 正常終了・異常終了・競合順序の単体試験。
  - 停止条件: 終了理由を列挙可能な型で表せない。

- [ ] H3-3 論理時計による時間切れと再送操作指示を実装する。
  - 完了条件: 実時間なしで期限設定、取消、期限切れを再現し、状態ごとの再送上限と中止理由を検査できる。
  - 含む: 時計値のドメイン型、期限切れ事象、タイマー操作指示。
  - 含まない: 実行基盤のタイマー、無条件の自動再接続。
  - 検証: 境界時刻、重複期限切れ、取消後期限切れの単体試験。
  - 停止条件: 仕様上の時間切れ値または再送規則を確定できない。

- [ ] H3-4 状態機械の性質試験で不正遷移と終了可能性を検査する。
  - 完了条件: 生成した事象列について不正状態を構築せず、終了事象後にデータ送信指示を返さず、失敗例を再現可能に縮小できる。
  - 含む: 決定論的な事象列生成と失敗例保存。
  - 含まない: PPP状態、外部I/O。
  - 検証: 通常検証に収まる性質試験と夜間の拡張試験。
  - 停止条件: 状態の不変条件を列挙できない。

### H3完了条件

H3-1からH3-4が完了し、SSTP制御接続の開始、維持、時間切れ、終了が純粋な事象と操作指示だけで再現できること。

理由: I/O接続前に状態遷移を固定すると、実環境の待ち時間や切断順序を決定論的に再現できるためです。

不採用理由: 非同期処理の中へ状態を埋め込むと、時間切れと切断の競合を単体試験できないためです。

## H4 HTTP・TLS通信路の縦切り

- [ ] H4-1 非同期実行基盤とTLS実装基盤を選定し、接続口と依存関係を記録する。
  - 完了条件: D-001とD-002が解決済みになり、二つ以上の実装候補、携帯端末制約、証明書検証、取消、保守状況、ライセンスを比較した決定が残る。
  - 含む: 必要最小限の依存追加と供給網記録。
  - 含まない: SSTP接続処理、汎用的なI/O trait群。
  - 検証: 依存構築、供給網検査、対象環境での最小接続口のコンパイル。
  - 停止条件: AppleまたはAndroidで採用候補を構築できない、あるいは証明書保管庫要件を満たせない。

- [ ] H4-2 TLS証明書とホスト名を検証してHTTP SSTP交渉を完了する。
  - 完了条件: ローカル試験サーバーに接続し、正しい証明書ではHTTP成功応答を得て、不正証明書、ホスト名不一致、不正HTTP応答を分類して拒否する。
  - 含む: 接続先ドメイン型、資格情報を含まない診断情報、取消可能な接続処理。
  - 含まない: SSTP制御メッセージ、プロキシ、自動再接続。
  - 検証: ローカル境界試験と証明書異常系試験。
  - 停止条件: 通常設定で証明書検証を無効化する必要が生じる。

- [ ] H4-3 通信路からSSTP状態機械を駆動し、制御接続手順を完了する。
  - 完了条件: ローカル試験サーバーとの断片化されたTLS I/Oで、HTTP交渉から接続肯定応答まで進み、状態機械の操作指示だけが副作用を要求する。
  - 含む: 有界I/O、取消、通信路切断の伝達、秘密を含まない診断事象。
  - 含まない: PPP認証、IPパケット転送、オペレーティングシステムVPN接続部。
  - 検証: ローカル端点間試験、断片化・結合・途中切断試験。
  - 停止条件: 純粋層へI/Oまたは実行基盤依存を入れる必要が生じる。

- [ ] H4-4 明示的に有効化する外部相互運用試験でSSTP制御接続を確認する。
  - 完了条件: 少なくとも一つの既存SSTPサーバーで接続要求・応答・正常切断を観測し、結果を再実行可能な報告として保存する。
  - 含む: `./scripts/verify-oracle` の実処理、環境不足時の明示的省略。
  - 含まない: 必須単体試験への外部サーバー依存、PPP認証成功。
  - 検証: 外部相互運用試験と保存済みの非秘密メタデータ。
  - 停止条件: 公開仕様と複数実装の挙動が矛盾し、互換方針の判断が必要になる。

### H4完了条件

H4-1からH4-4が完了し、証明書検証を省略せず、HTTP・TLS上で既存サーバーとのSSTP制御接続と正常切断を行えること。

理由: ローカルで再現可能な通信路試験を先に置き、環境依存の相互運用試験を独立させるためです。

不採用理由: 最初から外部サーバーだけで試験すると、証明書、HTTP、通信形式、状態遷移のどこで失敗したか特定しにくいためです。

## H5 PPP、認証、Crypto Binding、データ経路

- [ ] H5-1 PPP責務境界を決定し、フレーム形式とLCP交渉を純粋な状態機械として実装する。
  - 完了条件: D-003が解決済みになり、Configure-Request/Ack/Nak/RejectとTerminateを固定試験値から処理でき、状態機械が型付き操作指示を返す。
  - 含む: RFC 1661の根拠、選択肢長と重複の検証。
  - 含まない: 認証、IPCP、外部PPPプロセス。
  - 検証: 通信形式試験、状態遷移試験、異常選択肢試験。
  - 停止条件: PTY、TTY、`pppd`を共通データ経路へ置く必要が生じる。

- [ ] H5-2 PAP認証を開発用相互運用経路として実装する。
  - 完了条件: 資格情報を狭い入力境界から受け取り、PAP要求・成功・失敗を処理し、秘密をログまたは通常エラーへ含めない。
  - 含む: RFC 1334の通信形式、秘密の寿命と消去方針。
  - 含まない: PAPを製品推奨既定にすること、安全でないTLS設定。
  - 検証: 固定試験値、認証失敗、秘密非出力試験、外部相互運用試験。
  - 停止条件: 資格情報を複製して長期保持する必要が生じる。

- [ ] H5-3 MS-CHAPv2の通信形式と応答計算を実装する。
  - 完了条件: RFC 2759の試験値についてChallenge、Response、Success、Failureが一致し、利用者名とチャレンジの正規化規則を検査できる。
  - 含む: 列挙可能な認証失敗、必要な暗号依存の供給網記録。
  - 含まない: MPPEデータ暗号化、Crypto Binding計算。
  - 検証: RFC試験値、改変応答、異常長、秘密非出力試験。
  - 停止条件: 保守されていない暗号依存または監査不能な外部コードが必要になる。

- [ ] H5-4 MS-CHAPv2鍵素材を導出し、秘密の所有権と寿命を型で制限する。
  - 完了条件: RFC 3079の試験値と一致し、鍵素材が認証成功時だけ生成され、切断時に消去され、表示用型へ渡らない。
  - 含む: Crypto Bindingへ渡す鍵素材の狭い接続口。
  - 含まない: PPPデータのMPPE暗号化、Compound MAC。
  - 検証: 固定試験値、失敗時非生成、消去経路の試験。
  - 停止条件: 鍵素材を公開基本型として広範囲へ露出する必要が生じる。

- [ ] H5-5 SHA-256 Crypto Bindingを実装する。
  - 完了条件: MS-SSTPと関連仕様の試験値について証明書ハッシュとCompound MACが一致し、改変値を拒否し、SHA-256を優先する。
  - 含む: nonce、鍵素材、証明書ハッシュの受け渡し、接続完了状態への遷移。
  - 含まない: SHA-1のみを通常既定とすること、検証省略設定。
  - 検証: 固定試験値、改変検出、鍵消去、既存サーバー相互運用試験。
  - 停止条件: TLS実装から必要な証明書情報を安全に取得できない。

- [ ] H5-6 IPCPでIPv4アドレスとDNS設定を交渉する。
  - 完了条件: RFC 1332に基づきConfigure-Request/Ack/Nak/Rejectを処理し、得られたIPv4・DNS設定を型付き操作指示として返す。
  - 含む: 重複、未指定、不正長、再交渉の試験。
